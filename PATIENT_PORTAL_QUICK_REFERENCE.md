# Patient Portal Quick Reference Guide

## 🎯 For Government Stakeholders & MoH

### What Was Built

A comprehensive patient portal enhancement system that layers **10 major features** on top of the existing AfyaAI TZA platform:

1. ✅ **Digital Receptionist** - All core tasks in ≤3 steps
2. ✅ **Digital Patient Onboarding** - Replace paperwork, reduce wait times
3. ✅ **In-App Guidance** - Whatfix-style help (offline-first)
4. ✅ **Multi-Channel Notifications** - Reduce no-shows
5. ✅ **Self-Help Center** - Patient problem-solving without calling clinics
6. ✅ **EHR Integration UX** - Invisible sync, human-friendly errors
7. ✅ **Analytics Dashboard** - Track adoption, identify friction (privacy-safe)
8. ✅ **Patient Feedback Loops** - Build trust, improve quality
9. ✅ **Adoption Barriers Countermeasures** - Address digital divide
10. ✅ **Privacy & Compliance UX** - PDPA/TMDA visible trust features

## 🚀 How to Use

### For Patients

1. **Access Portal Hub**
   - Look for icons on dashboard
   - Tap to see all services in one place

2. **Complete Registration**
   - Follow 4-step guided process
   - Progress auto-saves
   - Can exit and return anytime

3. **Get Help**
   - Tap floating help button (blue circle with ?)
   - Search for answers
   - Chat with AI assistant
   - Contact support if needed

4. **Manage Notifications**
   - Tap notification bell (top right)
   - See all reminders and alerts
   - Take action directly from notifications

### For CHWs

- Monitor patient onboarding completion

- Guide patients through digital tools
- Report issues to MoH

### For MoH Administrators

- Review analytics dashboard (coming in production)

- Monitor system performance

## 📱 Features Overview

### Digital Receptionist (PatientPortalHub)
**Purpose**: One-stop shop for all patient services  
**Key Features**:
- 7 quick action cards
- Step count for each task
- Recent activity feed
- Appointment reminders
- Emergency contacts

**How to Access**: From patient dashboard

### Digital Onboarding (DigitalPatientOnboarding)
**Purpose**: Replace paper forms, reduce wait times  
**Key Features**:
- 4-step guided flow
- Auto-save (works offline)
- Progress tracking
- Consent education
- Completion celebration

**How to Access**: Auto-triggered for new patients

### In-App Guidance (InAppGuidanceSystem)
**Purpose**: Help patients without leaving the app  
**Key Features**:
- Floating help button
- Contextual tooltips
- Task checklist
- Interactive walkthroughs
- Offline help library

**How to Access**: Tap floating ? button (bottom right)

### Notifications (NotificationSystem)
**Purpose**: Reduce no-shows, improve adherence  
**Key Features**:
- Appointment reminders
- Medication reminders
- Form completion alerts
- Multi-channel (in-app, SMS, CHW)
- Priority levels

**How to Access**: Tap notification bell (top right)

### Self-Help Center (SelfHelpCenter)
**Purpose**: Reduce support calls, empower patients  
**Key Features**:
- Searchable FAQ
- AI chatbot (non-clinical)
- Category browsing
- Feedback on articles
- Support escalation

**How to Access**: Through in-app guidance system

## ❓ Common Questions

**Q: Do patients need smartphones?**  
A: Smartphone recommended, but USSD/SMS fallback available for feature phones.

**Q: What if patient has no internet?**  
A: All features work offline. Data syncs when connection available.

**Q: Is data secure?**  
A: Yes. Encrypted, PDPA-compliant, government-approved standards.

**Q: Can content be edited without redeployment?**  
A: Ready for CMS integration. Content can be admin-editable in production.

**Q: How are no-shows reduced?**  
A: Multi-channel reminders (in-app, SMS, CHW alerts) with action buttons.

**Q: What languages are supported?**  
A: Kiswahili (primary) and English (secondary). More languages can be added.

**Q: Does this replace clinic staff?**  
A: No. It supports staff by reducing paperwork and phone calls.

**Q: Can this scale nationwide?**  
A: Yes. Designed for government deployment at national scale.
